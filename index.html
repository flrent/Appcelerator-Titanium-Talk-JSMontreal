Appcelerator Titanium SDK

  • Simple idea : same code, multiple platforms
  • Code most people master : web development
  • Currently supports iOS, Android, Blackberry, and W8/WP8 in early 2013
  • Desktop version exists, TiDesk

How does Titanium really work ?

  1. You write JavaScript code
  2. Titanium API takes your JS code and links it with native functions
  3. At build time, Titanium API packages only what you need
  4. At runtime, your code is evaluated and native functions are called

Why it's so great ?

  1. Really fast to get started, on multiple platforms
  2. Native user experience, native UIs
  3. Rich SDK with 2D/3D animations, APIs modules(Facebook, 4square, Twitter..)
  4. Performance over mobile web
  5. It's JavaScript

Why it's not so great ?

  1. You don't really have access to the real code
  2. Even if the SDK is rich, it's hard to get out from the framework
  3. Appcelerator maintains it - a company : some features needs a subscription
  4. It's JavaScript

So when can you use Titanium ?

  • You want to write a simple (= non complex algorithms, graphics managements) app
  • You don't want to learn a native language/framework
  • Web mobile is good, but you want native feel for your users
